## Codex Obsidian Sync

- Treat this vault as the canonical place for cross-session project and job tracking.
- For any meaningful project, codebase, client engagement, or job/application workflow, update the vault by default unless the update would be pure noise.
- Obsidian updates are part of normal close-out for substantial work.
- For meaningful software/project work, prefer adding or updating notes under `Dev/Projects/`.
- For job search or application work, prefer adding or updating notes under `Career/Job Applications/`.
- Keep updates selective and high-signal: current status, outcomes, next steps, blockers, key dates, and links to repos/files/PRs.
- Do not create notes for trivial one-off actions.
- If the vault repo has unrelated local changes, leave them alone and only add the note relevant to the current task.
